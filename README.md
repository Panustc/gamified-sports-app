## 产品定位

**一句话描述**：一款面向大学生的游戏化运动激励工具，通过「六维角色养成 + 社交羁绊 + AI 动态干预」三引擎驱动，让运动像打游戏一样上瘾。

**目标用户画像**：
- 缺乏运动动力的普通大学生
- 需要社交陪伴的运动新手
- 有竞争心的校园运动爱好者

**核心设计理念**：
- 降低运动心理门槛（最小承诺：2 分钟拉伸即可保住积分）
- 用 RPG 成长感替代枯燥的重复
- 社交绑定制造"不运动就有损失"的轻度压力

## 游戏化系统设计

### 六维属性体系

```
STR 💪 力量 — 力量训练、举铁
AGI ⚡ 敏捷 — 跑步、跳绳、球类
STA ❤️ 耐力 — 游泳、骑行、长跑
MEN 🧠 精神 — 散步、冥想、瑜伽
CHA ✨ 魅力 — 舞蹈、团体运动、飞盘
LUK 🍀 幸运 — 天赋加成、特殊事件
```

- 属性初始值 10，上限 100
- 每次完成对应类型运动，相关属性 +1~4 点
- 属性影响 Boss 伤害、经验获取等

### 稀有度装备系统

| 稀有度 | 概率 | 特点 |
|--------|------|------|
| N | 50% | 基础属性 +1~2 |
| R | 30% | 属性 +3~5 |
| SR | 18.5% | 属性 +6~8，额外效果 |
| SSR | 1.5% | 全属性 +8~12，经验加成 |

- 10 连保底：若前 9 次未出 SR/SSR，第 10 次概率提升至 30%
- SSR 自带全息流光动画

### 赛季通行证

- 50 级赛季进度，每 30 经验升一级
- Lv.15：食堂减脂餐 8 折券
- Lv.30：限定动态头像
- Lv.50：绝版 3D 虚拟跑鞋

## AI 与个性化系统

### 动态任务生成

根据用户画像智能生成 3 个今日待办：

| 画像特征 | 任务示例 |
|----------|----------|
| 早八满课 + 宿舍偏好 | 5 分钟 Tabata、靠墙静蹲 |
| 考研冲刺 + 夜跑偏好 | 夜间 2km 慢跑、睡前拉伸 |
| 力量举铁偏好 | 上肢力量、核心训练 |
| 科研泡馆 | 校园 Citywalk、饭后百步走 |
| 潮流运动偏好 | 飞盘局、核心平衡特训 |

### 流失预测模型(模拟)

**输入特征**：
- 连续打卡天数(权重 45%)
- 天气因子(权重 20%)
- 期末课表密度(权重 35%)

**风险等级**：
- < 30%：状态良好
- 30-70%：中等风险 → 触发对赌契约干预
- > 70%：高风险 → 强制契约

### 搭子匹配算法

基于欧几里得距离计算用户间属性相似度：

```
similarity = (1 - sqrt(sum((u_i - p_i)²) * 0.7 + |lv_u - lv_p|/10 * 0.3)) * 100
```

## 社交系统

### 损失厌恶契约机制

当检测到用户流失风险升高时，触发对赌契约：
- **契约条款**：完成 2 分钟拉伸即可保住本周积分
- **违约惩罚**：扣除 300 XP + 失去本周连签奖励
- **心理学依据**：前景理论——人们对损失的敏感度远高于收益

### LBS 领地涂色战

设计思路：
- 将真实校园地理空间虚拟化为 3×3 领地格
- 院系竞争制造群体归属感
- 占领需真实运动贡献（GPS 轨迹验证模拟）

### 寝室 PK

- 步数数据绑定连签天数（`8723 + sk * 300`）
- 实时可视化对比柱状图

### 世界 Boss

- 全服玩家共同攻击"期末 DDL 狂魔"
- 总 HP 100,000，每次攻击基于力量属性造成伤害
- 每 12 秒 Boss 自动掉血，模拟真实战斗节奏

## 人文关怀设计

- 切换为坐姿/上肢专属训练流派
- 任务类型：轮椅校园漫步、上肢弹力带抗阻、坐姿核心控制

## 技术架构

### 整体架构

```
┌─────────────────────────────────────┐
│        single-page application       │
│  ┌───────────────────────────────┐  │
│  │   HTML (语义化结构)            │  │
│  │   CSS  (CSS变量 + 主题系统)     │  │
│  │   JS   (IIFE + 模块化函数)      │  │
│  └───────────────────────────────┘  │
│           localStorage                │
│         (离线持久化存储)              │
└─────────────────────────────────────┘
```

### 关键设计

| 方面 | 实现方式 |
|------|----------|
| 单页应用 | 纯 HTML/CSS/JS，无框架依赖 |
| 主题系统 | CSS 自定义属性(`:root` + `.theme-day`/`.theme-night`) |
| 数据持久化 | `localStorage` 序列化全部游戏状态 |
| Canvas 渲染 | 雷达图、折线图、对比雷达图均使用 Canvas 2D API |
| DPI 适配 | `window.devicePixelRatio` + `ctx.setTransform()` 缩放 |
| 动画 | CSS `@keyframes` + `transition` + `element.animate()` |
| 图标 | Font Awesome 6 + Unicode Emoji |
| 响应式 | `max-width: 480px` 手机布局，大屏手机外壳效果 |

### Canvas 可视化

| 图表 | 技术特点 |
|------|----------|
| 属性雷达图 | 6 维径向图、虚线坐标轴、动态缩放、发光描边 |
| 运动月报 | 30 天贝塞尔平滑曲线、渐变色填充、呼吸灯最高点标记 |
| 对比雷达图 | 双人属性对比、颜色区分、小尺寸自适应 |

### 主题系统变量

根变量定义全量色板，主题切换时仅需覆盖 `--bg`、`--text` 等核心变量，所有组件自动适配。

## 数据体系

### 核心数据模型

```javascript
{
  u: { nn, lv, xp, sk, ti, ch, cl, bu[], tc[], tk, av, st, recovery, access },
  st: { str, agi, sta, men, cha, luk },
  on: { g, p[], l, rd[] },
  gi: [{ i, n, r, c, b, v, bd }],
  eq: { i, n, r, c, b, v, bd },
  bo: { hp, dl[] }
}
```

### 经验公式

```
升级所需经验: xpL(Lv) = 50 + Lv × 30
打卡经验: base 25-40，SSR装备附加15%
```

### 卡路里体系

```
单次运动消耗: rd(200, 400) 千卡
月报模拟: 基准 500 ± 200(正弦波动)
请假成本: 500 千卡
```
